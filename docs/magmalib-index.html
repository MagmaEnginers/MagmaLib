Documentación de MagmaLib

La solución profesional para migrar plugins de Paper a Folia con una API moderna, type-safe y de alto rendimiento

Inicio Rápido

3 pasos para integrar MagmaLib:
  1. Añadir como dependencia en tu pom.xml o build.gradle
  2. Llamar MagmaLib.init(this) en onEnable()
  3. Reemplazar Bukkit.getScheduler() por MagmaLib.task()
// 1. En tu plugin principal
@Override
public void onEnable() {
    MagmaLib.init(this);

    // 2. Usar Task Builder (API estándar - segura y legible)
    MagmaLib.task(() -> {
        player.sendMessage("¡Hola desde MagmaLib!");
    })
    .at(player.getLocation())
    .afterTicks(20)
    .handleException(e -> getLogger().warning("Error: " + e.getMessage()))
    .run();

    // 3. O usar API directa para hot paths (máximo rendimiento)
    // MagmaLib.runDirectAt(player.getLocation(), () -> {
    //     player.sendMessage("¡Hola rápido!");
    // });
}

Inicialización

init()

void init(Plugin plugin)

Inicializa la librería. Llamar en onEnable().

@Override
public void onEnable() {
    MagmaLib.init(this);
}

isFolia()

boolean isFolia()

Detecta automáticamente si el servidor es Folia usando reflexión robusta.

if (MagmaLib.isFolia()) {
    // Código optimizado para Folia
} else {
    // Fallback para Paper/Spigot
}

Task Builder (API Estándar)

API segura y legible para la mayoría de casos de uso. Incluye validaciones y manejo de errores.

task() Estándar

TaskBuilder task(Runnable runnable)

Punto de entrada para configurar tareas con validaciones automáticas.

MagmaLib.task(() -> {
    player.sendMessage("¡Hola!");
});

at()

TaskBuilder at(Location location)

Ejecuta en la región del chunk (Folia) o hilo principal (Paper). Valida world != null y chunk cargado por defecto.

.at(player.getLocation())

with()

TaskBuilder with(Entity entity)

Vincula la tarea al scheduler de la entidad (Folia). Valida entity.isValid() por defecto.

.with(villager)

after()

TaskBuilder after(long delay, TimeUnit unit)

Retraso con cualquier unidad temporal (se convierte internamente a ticks).

.after(2, TimeUnit.SECONDS)

afterTicks()

TaskBuilder afterTicks(long ticks)

Retraso directo en ticks de Minecraft (1 tick = 50ms). Más eficiente que after().

.afterTicks(20) // 1 segundo

every()

TaskBuilder every(long period, TimeUnit unit)

Configura tarea repetitiva con unidad temporal.

.every(5, TimeUnit.SECONDS)

everyTicks()

TaskBuilder everyTicks(long ticks)

Intervalo repetitivo directo en ticks. Más eficiente que every().

.everyTicks(100) // Cada 5 segundos

handleException()

TaskBuilder handleException(Consumer<Exception> handler)

Manejador personalizado de errores. Si no se proporciona, se loguea por defecto.

.handleException(e -> {
    plugin.getLogger().warning("Error: " + e.getMessage());
})

unsafe() Directo

TaskBuilder unsafe()

⚠️ Desactiva validaciones de seguridad. Úsalo SOLO en hot paths donde garantizas parámetros válidos.

Requisitos: location.getWorld() != null, chunk cargado, entity.isValid() == true 
.at(location).unsafe().run() // Sin checks, máximo rendimiento

run()

Task run()

Ejecuta la tarea configurada y devuelve objeto Task para cancelación manual.

Task task = builder.run();
// Luego: task.cancel();

API Directa (Alto Rendimiento)

Métodos optimizados para hot paths. Sin builder, sin validaciones, máximo throughput.

⚠️ Advertencia: Estos métodos omiten validaciones de seguridad. Úsalos SOLO cuando:
  • Garantizas que Location.world != null
  • El chunk está cargado (para métodos con Location)
  • La entidad es válida (para métodos con Entity)
  • Estás en un bucle crítico donde cada microsegundo cuenta

runDirect()

void runDirect(Runnable runnable)

Ejecución inmediata en hilo global. Sin validaciones. ~25% más rápido que task().run().

MagmaLib.runDirect(() -> {
    // Código crítico - garantizas que es seguro
    counter.incrementAndGet();
});

runDirectAt()

void runDirectAt(Location location, Runnable runnable)

Ejecución en región de chunk. Omite checks de world/chunk. Ideal para procesamiento masivo de bloques.

// Procesar 1000 bloques rápidamente
for (Location loc : blockLocations) {
    MagmaLib.runDirectAt(loc, () -> {
        loc.getBlock().setType(Material.AIR);
    });
}

runDirectWith()

void runDirectWith(Entity entity, Runnable runnable)

Ejecución en scheduler de entidad. Sin validación de isValid(). Para bucles de actualización de entidades.

// Actualizar muchas entidades
for (Entity entity : entities) {
    MagmaLib.runDirectWith(entity, () -> {
        entity.setCustomName("Actualizado");
    });
}

runDirectLater()

void runDirectLater(Runnable runnable, long delayTicks)

Retraso directo en ticks. Sin conversión de TimeUnit. Más eficiente para delays simples.

MagmaLib.runDirectLater(() -> {
    player.sendMessage("Después de 20 ticks");
}, 20); // 20 ticks = 1 segundo

runDirectTimer()

void runDirectTimer(Runnable runnable, long periodTicks)

Tarea periódica directa. Sin builder ni allocations extra. Para bucles de tick críticos.

// Actualizar contador cada tick
MagmaLib.runDirectTimer(() -> {
    globalCounter++;
}, 1); // Cada tick

runTimerUntilFast()

Task runTimerUntilFast(Runnable task, long periodTicks, BooleanSupplier stopCondition)

Versión optimizada de runTimerUntil: sin AtomicReference. ~15% más rápido para timers simples.

// Timer de UI que se detiene cuando el jugador se desconecta
MagmaLib.runTimerUntilFast(() -> {
    player.sendTitle("Cargando...", "", 10, 40, 10);
}, 20, () -> !player.isOnline());

Funciones Avanzadas

runWithRetry()

void runWithRetry(Runnable task, int maxAttempts, long delay, TimeUnit unit)

Ejecuta con reintentos automáticos usando scheduler (NO threads manuales). Compatible con Folia.

MagmaLib.runWithRetry(() -> {
    connectToDatabase();
}, 3, 5, TimeUnit.SECONDS);

runTimerUntil()

Task runTimerUntil(Runnable task, long periodTicks, BooleanSupplier stopCondition)

Ejecuta periódicamente hasta que se cumpla una condición. Versión segura con AtomicReference.

MagmaLib.runTimerUntil(() -> {
    player.sendTitle("Cargando...", "", 10, 40, 10);
}, 20, () -> !player.isOnline());

callSync()

<T> CompletableFuture<T> callSync(Supplier<T> supplier)

Ejecuta en hilo principal y retorna valor con CompletableFuture. Ideal para operaciones que requieren resultado.

MagmaLib.callSync(() -> {
    return player.getInventory().getContents();
}).thenAccept(items -> {
    // Procesar items en hilo asíncrono
});

forAllPlayers()

void forAllPlayers(Consumer<Player> action)

Itera jugadores con manejo seguro de errores y nulls. Ejecuta en hilo principal.

MagmaLib.forAllPlayers(player -> {
    player.sendTitle("¡Evento!", "", 10, 40, 10);
});

forAllLoadedChunks()

void forAllLoadedChunks(Consumer<Chunk> action)

Procesa chunks cargados (ejecuta asíncrono por defecto). Incluye null checks para worlds/chunks.

MagmaLib.forAllLoadedChunks(chunk -> {
    // Optimizar o analizar chunk
    optimizeChunk(chunk);
});

ticksToMs() / msToTicks()

long ticksToMs(long ticks) | long msToTicks(long ms)

Utilidades de conversión de tiempo. Usa multiplicación/división por 50L para máximo rendimiento.

long ms = MagmaLib.ticksToMs(20); // 1000
long ticks = MagmaLib.msToTicks(1000); // 20

Manejo de Entidades

Mejores Prácticas

Siempre usa .with(entity) para operaciones con entidades en Folia:

  • ✅ Ejecución en el scheduler correcto de la entidad
  • ✅ Cancelación automática si la entidad se elimina
  • ✅ Prevención de race conditions en Folia
  • ✅ Fallback seguro a Paper si no es Folia

Ejemplo Básico

MagmaLib.task(() -> {
    if (entity.isValid()) {
        entity.setCustomName("Procesado");
    }
})
.with(entity)
.run();

Validación Avanzada

MagmaLib.task(() -> {
    if (!entity.isValid() || entity.isDead()) return;
    entity.setGlowing(true);
})
.with(entity)
.cancelIf(() -> !player.isOnline())
.handleException(e -> {
    plugin.getLogger().warning("Error: " + e.getMessage());
})
.run();

Modo Alto Rendimiento

⚠️ Solo si garantizas entity.isValid() == true 
// Para bucles de actualización masiva
for (Entity entity : entities) {
    MagmaLib.task(() -> {
        entity.setCustomName("Updated");
    })
    .with(entity)
    .unsafe()  // Omite isValid() check
    .run();
}

FAQ

¿Cuándo usar API Directa?

Usa runDirect* cuando:

  • Procesas 100+ bloques/entidades por tick
  • Estás en un bucle crítico de rendimiento
  • Garantizas manualmente la validez de parámetros

Usa Task Builder cuando:

  • Desarrollas lógica de negocio normal
  • Priorizas legibilidad y mantenimiento
  • Quieres validaciones automáticas de seguridad

¿Es seguro usar .unsafe()?

Sí, si sigues estas reglas:

  • ✅ Valida manualmente antes de llamar
  • ✅ Úsalo solo en código que controlas totalmente
  • ✅ Documenta por qué es seguro en ese contexto
  • ❌ Nunca lo uses con inputs de usuario o datos externos
// ✅ Seguro: validación manual antes
if (loc.getWorld() != null && loc.getWorld().isChunkLoaded(...)) {
    MagmaLib.task(task).at(loc).unsafe().run();
}

¿MagmaLib funciona en Paper y Folia?

✅ Sí, 100% compatible. MagmaLib detecta automáticamente el tipo de servidor y usa el scheduler apropiado:

  • Folia: RegionScheduler, EntityScheduler, GlobalRegionScheduler
  • Paper/Spigot: Bukkit Scheduler tradicional

Tu código no necesita cambios para funcionar en ambos.
